1. Two Sum

Objective:

Given an array of integers nums and an integer target, return indices of the two numbers such that they add up to target. You may assume that each input would have exactly one solution, and you may not use the same element twice.

Test Cases:

Example 1:
Input: nums = [2,7,11,15], target = 9
Output: [0,1]
Explanation: nums[0] + nums[1] = 2 + 7 = 9

Example 2:
Input: nums = [3,2,4], target = 6
Output: [1,2]
Explanation: nums[1] + nums[2] = 2 + 4 = 6

Example 3:
Input: nums = [3,3], target = 6
Output: [0,1]
Explanation: nums[0] + nums[1] = 3 + 3 = 6

Constraints:

  • 2 ≤ nums.length ≤ 104
  • -109 ≤ nums[i] ≤ 109
  • -109 ≤ target ≤ 109
  • Only one valid answer exists.

Real-World Scenarios:

The Two Sum problem models situations where you need to find two items that together meet a specific criterion. Examples include:

  • Budgeting: Selecting two expenses that sum up to a specific budget limit.
  • Shopping: Choosing two products to fully utilize a gift card's value.
  • Logistics: Combining two shipments to maximize cargo space without exceeding weight limits.
  • Diet Planning: Combining foods to meet a specific caloric intake.
  • Event Seating: Pairing guests to fill tables based on seating capacities.

2. Reverse String

Objective:

Write a function that reverses a string. The input string is given as an array of characters s. You must do this by modifying the input array in-place with O(1) extra memory.

Test Cases:

Example 1:
Input: s = ["h","e","l","l","o"]
Output: ["o","l","l","e","h"]

Example 2:
Input: s = ["H","a","n","n","a","h"]
Output: ["h","a","n","n","a","H"]

Example 3:
Input: s = ["A"]
Output: ["A"]

Constraints:

  • 1 ≤ s.length ≤ 105
  • s[i] is a printable ASCII character.

Real-World Scenarios:

Reversing a string is a fundamental operation with applications in various domains:

  • Palindromic Testing: Checking if a string reads the same backward as forward.
  • Data Serialization: Reversing data sequences in communication protocols.
  • Text Processing: Implementing undo features in text editors.
  • Cryptography: Simple encryption techniques that involve reversing strings.
  • UI Development: Displaying text in right-to-left languages.

3. Valid Anagram

Objective:

Given two strings s and t, return true if t is an anagram of s, and false otherwise.

An Anagram is a word or phrase formed by rearranging the letters of a different word or phrase, typically using all the original letters exactly once.

Test Cases:

Example 1:
Input: s = "anagram", t = "nagaram"
Output: true

Example 2:
Input: s = "rat", t = "car"
Output: false

Example 3:
Input: s = "aacc", t = "ccac"
Output: false

Constraints:

  • 1 ≤ s.length, t.length ≤ 5 * 104
  • s and t consist of lowercase English letters.

Real-World Scenarios:

Anagram detection has practical uses in several fields:

  • Cryptography: Deciphering codes where letters are scrambled.
  • Spell Checkers: Suggesting correct words when letters are jumbled.
  • Games: Word puzzles like Scrabble or word jumbles.
  • Natural Language Processing: Text normalization and analysis.
  • Genetics: Comparing DNA sequences that are permutations of each other.

4. Implement strStr()

Objective:

Implement strStr(). Given two strings needle and haystack, return the index of the first occurrence of needle in haystack, or -1 if needle is not part of haystack.

Clarification: What should we return when needle is an empty string? For the purpose of this problem, return 0 when needle is an empty string.

Test Cases:

Example 1:
Input: haystack = "hello", needle = "ll"
Output: 2

Example 2:
Input: haystack = "aaaaa", needle = "bba"
Output: -1

Example 3:
Input: haystack = "", needle = ""
Output: 0

Example 4:
Input: haystack = "mississippi", needle = "issip"
Output: 4

Constraints:

  • 0 ≤ haystack.length, needle.length ≤ 5 * 104
  • haystack and needle consist of only lower-case English characters.

Real-World Scenarios:

String searching algorithms like strStr() are essential in many applications:

  • Text Editors: Implementing the "find" functionality to search for substrings.
  • Search Engines: Matching query terms within web pages.
  • DNA Sequencing: Finding specific gene sequences within DNA strands.
  • Plagiarism Detection: Identifying copied content by searching for matching text segments.
  • Log Analysis: Searching for specific error codes or messages within logs.

5. Best Time to Buy and Sell Stock

Objective:

You are given an array prices where prices[i] is the price of a given stock on the ith day. You want to maximize your profit by choosing a single day to buy one stock and choosing a different day in the future to sell that stock. Return the maximum profit you can achieve from this transaction. If you cannot achieve any profit, return 0.

Test Cases:

Example 1:
Input: prices = [7,1,5,3,6,4]
Output: 5
Explanation: Buy on day 2 (price = 1) and sell on day 5 (price = 6),
profit = 6 - 1 = 5.

Example 2:
Input: prices = [7,6,4,3,1]
Output: 0
Explanation: In this case, no transactions are done, and the max profit = 0.

Example 3:
Input: prices = [2,4,1]
Output: 2
Explanation: Buy on day 1 (price = 2) and sell on day 2 (price = 4),
profit = 4 - 2 = 2.

Constraints:

  • 1 ≤ prices.length ≤ 105
  • 0 ≤ prices[i] ≤ 104

Real-World Scenarios:

This problem simulates scenarios where you need to maximize profit over time:

  • Stock Trading: Determining the optimal times to buy and sell stocks for maximum gain.
  • Commodity Markets: Buying and selling commodities like gold or oil at the best prices.
  • Real Estate: Investing in property when prices are low and selling when they peak.
  • Currency Exchange: Exchanging currencies at optimal times to benefit from fluctuating exchange rates.
  • Inventory Management: Purchasing goods at lower prices and selling them when market demand increases.
